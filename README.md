# 高考报考知识库（Gaokao Knowledge Base）

> 河南 · 新高考（3+1+2）· 历史类（纯文科）· 2026届

一个**可维护、可扩展、可追溯、可被程序调用**的高考报考知识库系统。

---

## 📖 项目背景

我是2026届河南高考生（新高考 3+1+2 模式），选科为历史+政治+地理。高考结束后，需要在出分前完成专业方向筛选、学校格局了解，出分后完成志愿填报。

市面上虽然有各种「志愿填报卡」和 AI 报考工具，但存在三个问题：

1. **数据不透明**：不知道分数推荐背后的依据是什么，来源不可追溯
2. **不够个性化**：无法结合自己的偏好做精细化筛选
3. **用完即弃**：出分那几天用完就扔了，无法持续维护和迭代

因此决定**自己动手**，构建一个结构化、可追溯、可长期维护的知识库。

## 🛠 自动化工具

### 招生章程爬虫

自动从阳光高考平台获取全国高校招生章程，并解析为结构化 Markdown。

```
91-脚本工具/招生章程爬虫/
├── step1_list_schools.py   # 获取全国高校列表
├── step2_fetch_charters.py # 批量下载招生章程全文
├── step3_parse_to_md.py    # 解析提取关键字段 → 结构化 Markdown
├── crawler_config.py       # 配置文件
└── requirements.txt        # Python 依赖
```

**技术栈**：Python 3.12 + requests + BeautifulSoup4

**使用方法**：
```bash
# 安装依赖
pip install -r requirements.txt

# 第1步：获取学校列表
python step1_list_schools.py --test   # 测试（100所）
python step1_list_schools.py          # 全量（约2800所）

# 第2步：下载章程
python step2_fetch_charters.py --limit 5          # 测试（5所）
python step2_fetch_charters.py --province 河南    # 只爬河南省高校
python step2_fetch_charters.py --all              # 全量爬取

# 第3步：解析生成 Markdown
python step3_parse_to_md.py           # 解析全部
python step3_parse_to_md.py --sync    # 解析后同步到知识库
```

## 🗺️ 使用路径

```
出分前（现在）
  │
  ├─→ 看「专业库」全景图 → 逐个读10个专业类 → 确定3-5个方向
  ├─→ 看「学校库」分层总览 → 了解985/211/双一流/省重点格局
  ├─→ 确认选科限制（新高考关键：历史+政史地能报什么）
  └─→ 确认特殊类型资格（国家专项/高校专项/公费师范生）

出分后
  │
  ├─→ 查分+位次+批次线
  ├─→ 对照录取数据定位能报的层次
  ├─→ 在对应层次中筛选学校（结合之前标记的兴趣方向）
  ├─→ 用冲稳保策略分配志愿
  ├─→ 在「我的意向」中填写候选方案
  └─→ 用查验清单逐项确认后提交
```
